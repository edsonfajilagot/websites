    ﻿ docs > ports > legoev3-output-port (kernel: 3.16.7-ckt11-5-ev3dev-ev3)

    General Info

    device_name legoev3-output-port
    port_name prefix1 out
    sysfs class lego-port
    # Modes 5

    Modes

    mode Description
    auto2 Automatically detect motors when they are connected.
    legoev3-motor3 Load the legoev3-motor device.
    rcx-motor4 Load the rcx-motor device.
    rcx-led5 Load the rcx-led device.
    raw6 Provide access to low level drivers.

    Notes

    1. The full port_name is in the formt:

      [<parent-port-name>:]<prefix><n>

      For example, if we are looking a port 1 of a sensor mux plugged into port 2 on the EV3, the port name might be in2:mux1. ↩

    2. Only the EV3/NXT large motors and the EV3 medium motor can be automatically detected. All other devices must be manually configured. ↩

    3. Configures the port to use the legoev3-motor driver module. The default driver is the EV3/NXT Large Motor (lego-ev3-l-motor). You can change the driver using the set_device attribute. ↩

    4. This can be use with MINDSTORMS RCX motors, Power Functions motors and any other ‘plain’ DC motor. By ‘plain’, we mean the motor is just a motor without any feedback. ↩

    5. This can be used with MINDSTORMS RCX LEDs, Power Functions LEDs or any other LED connected to pins 1 and 2 of the output port. ↩

    6. Exports gpios, pwm and analog/digital converter values to sysfs so that they can be controlled directly. ↩
